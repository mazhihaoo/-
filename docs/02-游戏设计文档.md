## 0. 一句话定位

**你是地震中的普通人，第一人称。凭本能做的"逃命"选择往往是致命的——你死了，镜头拉出来慢放告诉你为什么，再来一次爬更高的楼。** 用恐惧把救命知识焊进玩家脑子。

## 1. 设计铁律（从研究来的，任何人不准违背）

| # | 铁律 | 为什么（一句） |
|---|---|---|
| 铁律1 | **恐惧只在「你死了」那一刻拉满**，思考抉择时必须克制 | 吓瘫了就选不动、记不住（研究束2倒U曲线） |
| 铁律2 | **每次死后必须立刻给"你本该怎么做"的可执行正解** | 没给正解，玩家会"防御性回避"——直接关游戏（束2元分析） |
| 铁律3 | **3D 揭示只给一句话 + "错vs对"动作对比，绝不堆旁支知识** | 高唤醒会清空旁支信息，只记得住死因主线（束2 weapon focus） |
| 铁律4 | **正确答案恒定，绝不随机**（随机的只能是危险源位置） | 答案随机=教玩家"逃生靠猜"=反科普害人（束4） |
| 铁律5 | **声音是第一恐惧武器**，优先于画面精度投入 | 几KB音效 > 高精度模型（束1） |
| 铁律6 | **8MB 红线**：不导入大图片/精模，几何体全代码生成 | 撑爆唯一原因就是有人拖大文件进来 |
| 铁律7 | **3D 揭示第一次必看、不可跳过** | 防止"刷分模式"吃掉学习（束3+D观点） |

## 2. 完整游戏流程

> **抉择方式（2026-05-30 订正·核心演进）**：玩家**不点 UI 选项**，而是**第一人称走到空间里的不同地点**（楼梯口 / 电梯 / 坚固桌子 / 门框 / 窗边），**靠近任一处按 E 确认 = 做出那个选择**。下面流程图里的"✗冲楼梯 / ✓钻桌下"都是玩家用脚走过去、按 E 确认的结果——**抉择是用脚做的，不是用鼠标点的**。

```
【开场字幕】→【平静·办公室】→ 地震突袭 ─┬─ 节点1：往哪躲？(限时)
 黑屏一行字     暖光,可自由看      轰鸣+      │   ✗冲楼梯/挤电梯/站门框 → 你死了
 "下午2:28,     建立"安全感"      镜头狂晃    │   ✓钻坚固桌下抓牢 ─────────┐
  你在20楼"     (恐惧低)          (恐惧↑)     │                          ↓
                                            ├─ 节点2：还在晃,冲出去吗？(限时)
                                            │   ✗立刻冲出 → 你死了       │
                                            │   ✓留在掩护下 ───────────┤
                                            │                          ↓
                                            └─ 节点3：震停了,怎么走？(限时)
                                                ✗坐电梯/原地等 → 你死了
                                                ✓走楼梯快撤 →【通关→下一层(更高更难)】

每个「你死了」走同一条管线（通用模板,不逐个定制）：
  你死了(恐惧拉满) → 黑屏静默3-5s(心跳渐弱,恐惧骤降) → 镜头拉出·上帝视角慢放
                                                       +红圈高亮"你刚做的错 vs 正确动作"
                                                       +一句话正解 → 回到该节点重选

倒计时(方案C)：超时不直接跳过,判定为"你愣住了"=一种致命结果(教"犹豫也会死")
```

> **3D 揭示分层（2026-05-30 订正）**：
> - **v1（当前 demo）**：简化版——镜头从第一人称拉出到上帝视角俯视 + 红圈高亮你的位置 + 致命点闪红 + 一句话正解。
> - **v2（进化目标）**：引入 **物理动态碰撞模型 + 人体模型**，360° 模拟"人具体是怎么被砸死的"，符合物理定律/公式——这是最初愿景"3D 演示山体滑坡/泥石流/板块位移"的高级进化，震撼与教育性拉满。
> - **v2 人物模型选型（2026-05-30 调研定）**：用**代码程序化生成的低多边形小人**（《纪念碑谷》式几何风），放进死亡揭示里被楼板砸——零体积守 8MB、和低模场景风格统一、可代码做"趴下/被砸"动画。**否决动漫美型**（VRoid/VRM 单个 5-50MB 必爆红线 + 精模塞进低模场景严重违和）。理由：第一人称里玩家看不到自己的身体，人物模型最值钱的落点就是揭示上帝视角（把红圈→换成小人被砸，恍然大悟翻倍）。

**恐惧节拍（严格执行铁律1-3）**：

| 玩家正在经历 | 恐惧 | 设计动作 |
|---|---|---|
| 平静办公室 | 低 | 暖光、可自由看，建立安全感对比 |
| 地震突袭+看选项 | 中（克制） | 晃动**降一档**、选项文字**固定不跟着晃**、始终清晰可读 |
| 点下选择→「你死了」 | 拉满 | 视野骤黑、轰鸣峰值、镜头猛震 |
| 黑屏静默缓冲 3-5s | 骤降 | 心跳声渐弱 + 一行字慢慢浮现（防止被当成卡死） |
| 3D揭示+正解 | 低·冷静 | 低噪音、只一句话 + 一个动作对比 |

## 3. 核心架构：场景生产线 + 爬塔

**一句话**：把"游戏大脑"和"关卡数据卡"彻底分开。一层楼 = 一张数据卡，换卡 = 换关，其他全复用。这就是爬塔几乎零成本、8MB 涨不了几 KB 的原因。

**关卡数据卡格式（人1 开工第一件事就要定死这个）**——示意：

```js
const Floor = {
  level: 7,                          // 第几层（决定难度倍率）
  intensity: 0.7,                    // 震感强度 0-1 → 驱动晃动/视野收缩/音量
  layout: 'office_window',           // 危险源布局（决定几何体摆放,可随机选）
  nodes: [
    {
      prompt: '往哪躲？',
      timeLimit: 8,                  // 倒计时秒数（层越高越短）
      options: [
        { label:'冲下楼梯',   result:'DEATH', cause:'stairwell_collapse', oneLineTruth:'地震时往楼下跑,是死亡率最高的选择' },
        { label:'挤进电梯',   result:'DEATH', cause:'elevator_drop',      oneLineTruth:'断电会把你困在电梯里' },
        { label:'钻桌下抓牢', result:'SURVIVE', /* 正确,进下一节点 */ },
        { label:'站到门框下', result:'DEATH', cause:'falling_debris',     oneLineTruth:'现代门框不比墙结实,挡不住落物' },
      ],
      onTimeout: { result:'DEATH', cause:'frozen', oneLineTruth:'地震中犹豫,本身就是致命的' }
    },
    /* 节点2、节点3 同结构 */
  ]
}
```

**大脑（状态机）读这张卡，驱动整个流程。** 第二层 = 填一张新卡（换 nodes 的 options/cause/truth + 调 intensity/timeLimit）。人2/3/4 全都"读这张卡"干活，互不打架。

**爬塔规则**：通关一层 → 解锁更高层 → 层越高：`intensity`↑（更晃）、视野收缩更狠、`timeLimit`↓（更急）、计分倍率↑。

## 4. 视觉 + 音效策略

**视觉三层预算**（代入感靠光影动态，不靠写实）：
- 远景/背景：极简低模 + 雾/压暗糊掉（几乎零成本）
- 全局质感：光照 + 电影后期滤镜（pmndrs/postprocessing 的 Bloom辉光/Vignette暗角/ChromaticAberration色差/Noise噪点）+ 统一冷调
- 近景焦点物：玩家盯着看的那几样（朝你倒的柜子、脚下裂缝、门把）砸细节

**微恐机制（纯代码，零资源 — 采纳同学观点C）**：
- **视野随震级 + 犹豫时间动态收缩**：只亮角色周围一圈，其余压暗。越愣越黑 = 倒计时C的视觉身体。**注意：收缩的是环境视野，选项文字始终清晰**。
- 倒计时**红光闪烁**，逼近黑暗时只剩**心跳声**。
- FOV：探索 55°，死亡回放拉出到 90° 制造"解脱感"对比。

**音效（铁律5，最便宜的恐惧武器）**：
- 氛围音（地鸣/风）用 **WebAudio 代码现场合成**（低频oscillator+噪声），零文件。
- 关键音效（玻璃碎裂/警报/心跳）用小体积文件或合成，按需取舍。
- 空间化：`THREE.PositionalAudio`，把声源放玩家身后制造"有东西靠近"。

**技术栈（拍板）**：Three.js + pmndrs/postprocessing + WebAudio。**Godot web 导出已排除**（空包就 33-56MB，必爆 8MB）。

## 6. 死亡循环 + 钩子（采纳同学观点D）

死亡界面 = **死因提示 + 本局分 vs 历史最高分 + 巨大"再次挑战"按钮**。
- 秒级重来（纯前端 reset state，无加载屏）。
- 分数奖励"做对的避险决策 + 爬的层数"，与科普对齐（爬越高=学越多）。
- 历史最高分存 `localStorage`（零体积）。

## 9. MVP 边界（2天，信条：边界纪律 + 骨架终态）

**架构按"可无限爬塔"一步到位；功能肌肉分期做：**
