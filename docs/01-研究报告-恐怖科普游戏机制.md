## 束1 — 恐怖游戏机制：怎么用最低成本吓到人

**核心结论：在 8MB 纯前端里，声音是性价比最高的恐惧杠杆，远超 3D 模型精度。**

1. **声音 > 画面**。方向性音频、战略性沉默、声画同步（尖锐音 + 灯光闪烁同帧）。WebAudio 是浏览器原生能力，几 KB 的音效就能制造比高精度模型更强的恐惧。— [Wayline: Silence is Scary](https://www.wayline.io/blog/silence-is-scary-sound-design-horror-games)
2. **未知 = 最大恐惧**。有效恐惧三要素 = 氛围 / 未知 / 脆弱感，让玩家想象力自己跑。限制视野、低光、低能见度。— [Game Developer: Three Factors of Effective Horror](https://www.gamedeveloper.com/design/the-three-factors-of-effective-horror-design)
3. **"安慰剂"机制**（Amnesia 创意总监亲述）：故意不告诉玩家惩罚的机械后果，靠画面晃动 / 呼吸音 / 视觉扭曲暗示"有惩罚"，玩家自己脑补恐惧。低成本 web 端最该抄。— [Game Developer: Amnesia's Sanity Meter](https://www.gamedeveloper.com/design/game-design-deep-dive-i-amnesia-i-s-sanity-meter-)
4. **张弛节奏**：jump scare 只能用一次；持久恐惧靠"给安全区 → 打破已建立的规则"反复释放张力。— [Game Developer: Balancing Act of Tension](https://www.gamedeveloper.com/design/the-balancing-act-of-tension-in-horror-game-design)
5. **视野限制（FOV 压缩）**：Layers of Fear 把 FOV 锁在 50-60°，威胁只在视野边缘触发。Three.js 里 `camera.fov=55`，死亡回放镜头拉出时扩到 90°，制造"上帝视角解脱感"对比。
6. **空间化音效**：`THREE.PositionalAudio` + WebAudio `PannerNode`，把低频音源放在玩家身后偏左，制造"有什么在靠近"的方向感。

## 束2 — 恐惧 vs 学习的脑科学：⚠️ 关键发现，改写设计

**核心结论（反直觉，修正了我们"怕太恐怖"的担心）：恐惧可以放心拉满，决定成败的不是恐惧强度，是死后有没有立刻给"你本该怎么做"的可执行动作。**

1. **最权威的恐惧诉求元分析（Tannenbaum 2015，127 研究 / 2.7 万人）：恐惧本身越强越有效，几乎没有"吓过头反而劝退"的实证。** 真正的开关是消息里有没有给出清晰、可执行的应对方法（efficacy / 自我效能感）。— [APA: Appealing to Fear Meta-Analysis](https://www.apa.org/pubs/journals/releases/bul-a0039729.pdf)
2. **强恐惧 + 高效能信息 → 最大行为改变；强恐惧 + 低效能（没给方法）→ 最大"防御性回避"**（玩家扭头、否认、关掉游戏）。死亡场景拉满威胁感，复活/揭示环节立刻补上"简单到必然学会"的正确做法。— [Witte & Allen 2000](https://regulatorwatch.com/wp-content/uploads/2020/06/Witte-and-Allen-2000-A-Meta-Analysis-of-Fear-Appeals-Implications-for-.pdf)
3. **高唤醒会窄化注意力到威胁源（weapon focus），记住死因主线，记不住旁支信息。** 所以 3D 揭示必须把"正确知识"死死绑定在死亡原因这条主线上，不塞旁支，否则被高唤醒清空。— [Weapon Focus Effect](https://en.wikipedia.org/wiki/Weapon_focus) / [Temporal Dynamics of Emotional Memory (PMC)](https://pmc.ncbi.nlm.nih.gov/articles/PMC1906714/)
4. **肾上腺素 = 生物荧光笔**：吓到死的那一刻是记忆编码最强窗口，此时立刻揭示正确逃生知识，留存率远高于平铺直叙。— [McGaugh, NCBI](https://www.ncbi.nlm.nih.gov/books/NBK3907/)
5. **耶基斯-多德森倒 U 调和**：任务越复杂、越陌生，最佳唤醒点越低。"思考逃生选项"是复杂任务 → 做抉择时恐惧要克制（留认知带宽）；恐惧峰值放在"抉择已锁定、揭晓你死了"那一刻。— [Psychology Town: Arousal Management](https://psychology.town/motivation-emotion/maximizing-learning-arousal-management-education/)

### 由此得出的"恐惧强度时序标定"（直接落地）
| 阶段 | 恐惧强度 | 为什么 |
|---|---|---|
| 平静建立 / 思考抉择 | **低-中**（克制） | 复杂决策需要认知带宽，吓瘫了就选不动 |
| 抉择锁定 → "你死了" | **拉满** | 记忆编码最强窗口，肾上腺素烙印死因 |
| 死后 3-5 秒黑屏静默 | **骤降** | 缓冲带，让理智脑（前额叶）重新上线 |
| 3D 揭示 + 正确知识 | **低**（冷静） | 低视觉噪音，只聚焦"正确动作 vs 你刚犯的错"，一句话结论 |

## 束3 — "你死了"循环：怎么让人想再来而不是弃坑

**核心结论：死亡是反馈不是惩罚；秒级重来 + 每次死揭示一块新真相 = 重玩变揭秘。**

1. **好的死 vs 坏的死**："你失败了，再试，且我知道为什么"（好）vs"你失败了，重做无聊部分"（坏）。死了没拿到有用信息，失败就从"教训"变成"浪费时间"。— [Death in Games Is More Than a Game Over Screen](https://www.fixgamingchannel.com/death-in-games-is-more-than-a-game-over-screen/)
2. **低重试摩擦**（roguelike 铁律）："每局可以几秒结束，每局可以几秒重开。"纯前端只需 reset state、无加载屏，天然适配。— [Indiecator: Roguelikes & Progression](https://indiecator.org/2022/03/30/on-roguelikes-and-progression-systems/)
3. **每次死推进揭秘**（Hades）：每次死亡喂给玩家新信息/新剧情，重复变成累积而非无聊。映射：每个"你死了"揭穿一个地震误区，重跑 = 揭秘。— [Hades 成功拆解](https://luke-shaw.medium.com/two-short-pieces-on-hades-and-its-narrative-and-mechanical-success-2e4633c5338c)
4. **"YOU DIED"是情绪重击**（Dark Souls）：死亡画面本身是标志性情绪节拍，把绝望翻转成"我为什么死？"的好奇——正是钩入 3D 揭示的扳机。— [Game Developer: Narrative Design in Dark Souls](https://www.gamedeveloper.com/design/narrative-design-in-dark-souls)

## 束4 — 地震逃生权威知识：✅ 多源核实完成（教错会害人）

**核心结论："生命三角"是被官方一致否定的伪知识，正好当我们游戏的反面靶子。官方正解 = Drop-Cover-Hold On。**

### 排雷：生命三角（Triangle of Life）= 伪知识
USGS / FEMA / CDC / 红十字会 / Global Earthquake Model / Earthquake Country Alliance **一致否定**。它基于两个错误假设：① 建筑地震必塌（错，发达国家罕见整体 pancake 塌）；② 塌时必压碎所有家具（错，人确实能在家具下幸存）。
— [USGS: Is Triangle of Life legitimate?](https://www.usgs.gov/faqs/what-triangle-life-and-it-legitimate) / [Wikipedia: Triangle of Life](https://en.wikipedia.org/wiki/Triangle_of_Life)

### 三个抉择节点的官方正解（一级源）
| 节点 | 大众直觉（致命错误 = 我们的弹药） | 官方正解 | 来源 |
|---|---|---|---|
| **① 剧烈晃动那一刻** | 冲下楼 / 挤电梯 / 站门框 | **趴下 Drop - 躲坚固桌下护头颈 Cover - 抓牢 Hold On**；别跑户外、别站门框 | [USGS: During an EQ](https://www.usgs.gov/faqs/what-should-i-do-during-earthquake) / [Ready.gov](https://www.ready.gov/earthquakes) |
| **② 还在晃，要不要冲出去** | 立刻冲出建筑 | **别冲出去**，飞落物是最可能致伤源，留在掩护下 | [FEMA: Don't Use a Doorway](https://community.fema.gov/ProtectiveActions/s/article/Earthquake-Personal-Cover-Barriers-Do-Not-Use-a-Doorway) |
| **③ 震停后撤离** | 坐电梯下楼 | **别用电梯**（断电困人 + 喷淋启动）；走楼梯，远离窗户和外墙 | [CDC: During an EQ](https://www.cdc.gov/earthquakes/safety/stay-safe-during-an-earthquake.html) |

**门框辟谣**（节点①常见错）：现代房屋门框不比墙体强，挡不住飞落物。谣言源自 1900 年代土坯房只剩门框的误读，1940-70 年代被写进教材。— FEMA 同上。

## 束5 — 技术栈选型：✅ 拍板 Three.js

**核心结论：Three.js + pmndrs/postprocessing + WebAudio。Godot web 导出直接排除（必爆 8MB）。**

| 引擎 | 空包体积 | 8MB 内？ | 备注 |
|---|---|---|---|
| **Three.js** ✅ | **~168KB gzip**（最轻） | ✅ 绰绰有余 | 后处理有现成库、WebAudio 原生、社区/中文资料最多、第一人称 demo 路径成熟 |
| PlayCanvas | ~150-170KB | ✅ | 开箱即用带编辑器+物理+音频，省搭架构；但资料不如 Three.js |
| Babylon.js | ~1.4MB | ⚠️ 太重 | 8MB 预算下吃掉近 1/5，不划算 |
| **Godot 4 web** ❌ | **33-56MB** | ❌ 必爆 | 空 2D 节点导出就 >40MB，还卡 CloudFlare/GitHub 25MB 单文件上限，优化吃掉黑客松时间 |
| 纯 WebGPU | 最小 | ✅ 但 | 太底层，2 天写不完 |

- 后处理（恐怖氛围核心）：[pmndrs/postprocessing](https://github.com/pmndrs/postprocessing) 提供 Bloom/Vignette 暗角/ChromaticAberration 色差/Noise 噪点，EffectPass 合并到单次全屏 pass 降开销，按需 import 体积友好。
- 选型对比源：[utsubo: Three vs Babylon vs PlayCanvas 2026](https://www.utsubo.com/blog/threejs-vs-babylonjs-vs-playcanvas-comparison) / [Godot web 体积 issue #68647](https://github.com/godotengine/godot/issues/68647) / [VIVERSE: Web 3D Engines Overview](https://docs.viverse.com/optimization/overview-of-3d-web-rendering-engines)

## 对设计的总影响（一句话提炼）

> **恐惧拉满不怕、但只在"你死了"那一刻拉满；死后立刻冷静下来，用一句话 + 一个 3D 对比动作把"你本该 Drop-Cover-Hold"钉进玩家脑子；秒级重来，每次死揭穿一个地震误区（生命三角/门框/坐电梯/往外跑）；全部用 Three.js 实现。**
