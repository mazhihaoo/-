## 一句话

第一人称地震逃生科普游戏。主轴：**玩家凭本能做反直觉的致命选择 → 「你死了」→ 3D 揭示原因 → 再来**。用恐惧把救命知识焊进脑子。黑客松 4 人 / 2 天 / 8MB 纯前端。

- 核心循环已通：第一人称 → WASD 走动 → 地震 → 走到某处按 E 抉择 → 你死了 → 简化 3D 揭示 → 再来 → 爬塔。
- **下一步候选**：完整流程打磨 / v2 物理人体模型揭示（见 `docs/02` §2 揭示分层）/ 团队四人分工开工（`docs/02` §7）。

## 不可妥协的约束

- **7 条设计铁律**见 `docs/02` §1（恐惧时序、死后必给可执行正解、答案恒定、声音第一、揭示不堆旁支等）。
- **8MB 红线**：几何体全代码生成，不导入大图片 / 精模 / 音频文件。撑爆的唯一原因就是有人拖大文件进来。
- **地震逃生知识必须照 `docs/02` §5 正解表**（已多源核实 USGS/FEMA/CDC），不准自己发挥——**教错会害人**。

## ⚠️ 重要历史教训（别重蹈覆辙）

- 玩法曾从"抉择冒险"**跑偏成"躲红方块的反应游戏"**（废弃存档在 `docs/_archive/`）。根因：把"上瘾"当成目标、用高频反应取代了科普抉择。
- **回正铁律：本游戏的吸引力靠「代入 + 抉择紧张 + 你死了的震撼 + 恍然大悟」，不是跳一跳式高频反应。别再往"加反应玩法"偏。**
- **代码教训**：`$ = getElementById` 不能配 CSS 选择器（`$('#x .y')` 会返回 null 然后崩），要用 `document.querySelector`——这个错犯过两次，揭示阶段崩溃就是它。

## 技术 / 怎么跑 / 怎么验证

- 纯前端单文件，Three.js（CDN）+ WebAudio 合成。
- **跑**：项目根 `python3 -m http.server 8137` → `http://localhost:8137/`（需 http，**不能双击 file://**，ES module + CDN 会被拦）。
- **验证**：可用 CDP（web-access skill）连本地 Chrome 自己截图验证。`index.html` 内置 `window.__dbg(cmd)` / `window.__g()` 调试钩子（开发期用，正式交付前删）。**注意**：后台标签页 rAF 被浏览器节流，依赖时间推进的动态逻辑（倒计时/近点判定）在后台跑不动，要么前台、要么用 `__dbg` 钩子手动触发 + 强制渲染。
